Grupo 1

Tooltips

Los tooltips ofrecen ayuda breve, normalmente un único renglón de texto, destinada a aclarar el propósito de un control o mostrar información contextual muy corta. Son ideales para iconos, botones o campos con espacio limitado.

Inicialización y selectores

En este proyecto se inicializan automáticamente con jQuery. La lógica principal es:

// inicializa tooltips, excluyendo elementos que son popovers
    $('[data-toggle="tooltip"], [title]:not([data-toggle="popover"])').each(function () {
        var $el = $(this);
        var $modal = $el.closest('.modal');
        var container = $modal.length ? $modal : 'body';
        $el.tooltip({ container: container });
    });
Por qué excluir popovers

Si un elemento tiene tanto tooltip como popover (o comparte title con un popover), el usuario vería dos capas de UI. Para evitar confusión, los elementos marcados como popover se excluyen del init de tooltips.

Container y z-index

Problema: cuando un control está dentro de un modal, el tooltip añadido al body puede quedar detrás del backdrop debido al stacking context y z-index. Solución: inicializar el tooltip con container apuntando al propio .modal si el elemento está dentro de uno. Esto lo posiciona dentro del mismo contexto de apilamiento que el modal y evita que quede oculto.

Accesibilidad y keyboard

Los tooltips deben activarse con hover y focus. Asegúrate de que los elementos sean accesibles por teclado (tabindex cuando corresponda) y contiene texto alternativo claro en title. Para ARIA, Bootstrap gestiona role="tooltip" y aria-describedby automáticamente al inicializar.

Opciones y personalización

Puedes pasar opciones: delay (ms), placement, template, html. Evita usar html: true sin sanitizar contenido.

Casos dinámicos y rendimiento

Si la página crea muchos elementos con tooltips dinámicamente, inicialízalos al crearlos o usa delegación. Evita inicializar cientos de tooltips simultáneamente: considera una estrategia que inicialice solo cuando el elemento entra en vista (intersection observer) o cuando el usuario interactúa.

Popovers

Los popovers muestran contenido más rico: título y cuerpo, y pueden contener enlaces, botones o incluso pequeñas formas. Son más intrusivos que los tooltips y se usan cuando hace falta más contexto o acciones.

Inicialización y fragmento de código

Se inicializan con un patrón similar al de los tooltips, y el container dinámico evita problemas dentro de modales:

$('[data-toggle="popover"]').each(function () {
        var $el = $(this);
        var $modal = $el.closest('.modal');
        var container = $modal.length ? $modal : 'body';
        $el.popover({ container: container, trigger: 'focus' });
    });
Triggers: focus vs click

En el proyecto usamos trigger: 'focus' para que el popover se cierre automáticamente al perder foco. 'click' es otra opción (toggle), pero puede dejar el popover abierto hasta que el usuario haga otra acción explícita; evalúa qué experiencia quieres ofrecer.

Contenido HTML y seguridad

Si necesitas HTML dentro del popover, activa data-html="true" y valida/sanitiza el contenido si proviene de usuarios. Bootstrap provee sanitización por defecto, pero conviene definir las etiquetas/atributos permitidos si se inserta HTML dinámico.

Comportamiento con múltiples popovers

Para evitar que múltiples popovers se queden abiertos, puedes cerrar otros popovers cuando se abre uno nuevo. Un patrón común:

$(document).on('click', function (e) {
        $('[data-toggle="popover"]').each(function () {
            // si el click no es dentro del popover ni en el disparador, ciérralo
            if (!$(this).is(e.target) && $(this).has(e.target).length === 0) {
                $(this).popover('hide');
            }
        });
    });
Accesibilidad

Asegúrate de que los disparadores sean focusables y que el popover tenga atributos ARIA adecuados (aria-describedby, manejo de escape key para cerrarlo). Prueba con lector de pantalla y navegación por teclado.

Uso con datos dinámicos

Para contenido cargado mediante AJAX, actualiza el contenido del popover y re-posiciónalo (llamando a popover('show') después de insertar el contenido). Ten en cuenta latencias y estados de carga.
